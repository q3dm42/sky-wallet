# Skypro Wallet

SPA-приложение для учета и анализа расходов. Проект собран на React + Vite, сверстан по макетам Skypro Wallet и подключен к API Skypro.

## Что реализовано

- Регистрация и вход через API с валидацией формы и отображением ошибок запроса.
- Сохранение токена и данных пользователя в `localStorage`; после обновления страницы авторизация сохраняется.
- Защищенные маршруты: страницы расходов и аналитики доступны только авторизованному пользователю.
- Единый API-слой в `src/api`: клиент запросов, методы авторизации, методы транзакций и нормализация ответов.
- Страница "Мои расходы" с загрузкой расходов с сервера, состоянием загрузки и обработкой ошибок.
- Добавление нового расхода через API с полями описания, категории, даты и суммы.
- Удаление расхода через API с подтверждением действия.
- Empty state для таблицы, когда у пользователя еще нет расходов.
- Страница "Анализ расходов" с запросом расходов за выбранный период и графиком по категориям.
- Общий контекст транзакций для загрузки, добавления, удаления и повторного получения расходов.
- Адаптивная верстка, единые категории, иконки, состояния кнопок и лоадеры.
- Проверки проекта настроены без предупреждений: `dist` и референсный проект исключены из ESLint.

## Страницы

- `/login` - вход.
- `/register` - регистрация.
- `/expenses` - список расходов и добавление расхода.
- `/analysis` - анализ расходов по периоду.

## API

По умолчанию приложение использует:

```text
https://wedev-api.sky.pro/api
```

Базовый URL можно переопределить через переменную окружения:

```bash
VITE_API_URL=https://example.com/api
```

Основные методы вынесены в отдельные файлы:

- `src/api/client.js` - общий `fetch`-клиент, токен авторизации и обработка ошибок.
- `src/api/authApi.js` - вход и регистрация.
- `src/api/transactionsApi.js` - получение, добавление, удаление и загрузка расходов за период.
- `src/api/helpers.js` - нормализация ответов и формат дат для API.

## Установка и запуск

```bash
npm install
npm run dev
```

После запуска приложение будет доступно по адресу, который покажет Vite в терминале.

## Проверка и сборка

```bash
npm run lint
npm run build
```

Обе команды должны завершаться без ошибок и предупреждений.
